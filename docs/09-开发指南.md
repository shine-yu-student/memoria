# 09 — 开发指南

## 环境要求

- Node.js >= 18
- npm >= 9

## 本地开发

```bash
# 安装依赖
cd memoria
npm install

# 启动开发服务器（默认 http://localhost:5173）
npm run dev

# 生产构建
npm run build

# 预览生产构建
npm run preview
```

## 项目命令

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动 Vite 开发服务器（热更新） |
| `npm run build` | TypeScript 类型检查 + Vite 生产构建 |
| `npm run preview` | 本地预览构建产物 |

## 扩展指南

### 添加新的语言模块

1. 在 `src/components/` 下创建新目录（如 `japanese/`）
2. 定义新的数据类型（在 `src/types/index.ts` 中添加）
3. 添加新的 storage 函数（在 `src/utils/storage.ts` 中添加）
4. 在 `src/App.tsx` 的 TabType 和导航中添加新 Tab
5. 文档同步更新到 `docs/` 目录

### 添加新的文章导入格式

目前支持手动粘贴文本和 JSON 导入。如需支持文件格式（如 `.txt`, `.docx`），可在 `ArticleManager.tsx` 中添加新的文件读取逻辑。

### 修改句子切分规则

编辑 `src/utils/splitter.ts` 中的 `DELIMITER_RE` 正则表达式。

### 修改比对逻辑

编辑 `src/utils/diff.tsx` 中的 `computeDiff()` 函数。

## 已知限制

- 所有数据存储在浏览器 localStorage 中，清除浏览器数据将丢失所有记忆内容
- 仅支持纯文本，不支持富文本或图片
- 不支持服务端同步或多设备同步
- 不适合存储超大量文章（localStorage 容量限制约 5-10MB）

## 构建部署

```bash
npm run build
# 输出目录: memoria/dist/
# 可将 dist/ 目录部署到任意静态文件服务器
```

## 代码规范

- 使用 TypeScript strict 模式
- 组件使用 `React.FC` 类型
- 样式使用内联 `style` 对象（保持零依赖）
- 所有数据操作走 `src/utils/storage.ts` 层
