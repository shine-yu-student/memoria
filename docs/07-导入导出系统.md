## 入口变化

导入导出功能已从独立页面迁移至**设置面板**的"数据"分类中。用户点击侧栏底部的 ⚙️ 设置按钮，在设置弹窗中选择"数据"分类即可进行全量导入导出。

单个资源的导出按钮保留在各管理页面中：
- **文章编辑页**：导出按钮位于标题右侧。
- **词书/句书编辑页**：导出按钮位于操作栏中。

## JSON 序列化

### 单资源导出

每篇文章/词书/句书都可单独导出为 JSON：

```json
{
  "id": "abc123...",
  "title": "文章标题",
  "content": "全文内容……",
  "sentences": ["句子1", "句子2"],
  "createdAt": 1700000000000
}
```

通过 `SingleExportBtn` 组件触发下载：

```tsx
<SingleExportBtn label="文章名" data={article} filename="article-标题.json" />
```

### 全量备份格式

```typescript
interface MemoriaData {
  version: 1;
  articles: Article[];
  wordBooks: WordBook[];
  sentenceBooks: SentenceBook[];
  exportedAt: number;
}
```

全量导入采用**合并模式**（非覆盖）：
- **文章**：按标题匹配，同名文章保留原有不覆盖。
- **词书/句书**：按标题匹配并合并条目，同英文内容去重。

## 文件处理

### 下载

使用 `Blob + URL.createObjectURL` 实现浏览器端文件下载：

```typescript
function downloadJSON(data: unknown, filename: string) {
  const blob = new Blob([JSON.stringify(data, null, 2)], { type: 'application/json' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = filename;
  a.click();
  URL.revokeObjectURL(url);
}
```

### 上传

使用 `FileReader` 读取文件内容并解析 JSON：

```typescript
function readJSONFile(file: File): Promise<unknown> {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onload = () => resolve(JSON.parse(reader.result as string));
    reader.onerror = () => reject(new Error('文件读取失败'));
    reader.readAsText(file);
  });
}
```

## 错误处理

- JSON 解析失败 → 弹出错误提示。
- 无效的数据格式 → 格式校验后拒绝导入。
- 文件读取失败 → 捕获异常并提示。
