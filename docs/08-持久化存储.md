# 08 — 持久化存储

## 存储方案

使用浏览器 `localStorage` 进行客户端持久化存储。所有数据以 JSON 字符串形式存储在浏览器的本地存储中。

## Key 命名规范

所有 key 使用 `memoria:` 前缀：

| Key | 存储内容 |
|-----|----------|
| `memoria:articles` | 文章列表（`Article[]`） |
| `memoria:wordBooks` | 词书列表（`WordBook[]`） |
| `memoria:sentenceBooks` | 句书列表（`SentenceBook[]`） |

## 存储层 API

文件：`src/utils/storage.ts`

### 通用工具

```typescript
function loadJSON<T>(key: string, fallback: T): T {
  try {
    const raw = localStorage.getItem(key);
    return raw ? JSON.parse(raw) : fallback;
  } catch {
    return fallback;
  }
}

function saveJSON<T>(key: string, data: T): void {
  localStorage.setItem(key, JSON.stringify(data));
}
```

## 边界情况

- **localStorage 不可用**：私有模式下的某些浏览器可能禁用 localStorage，代码使用 try-catch 捕获异常，返回空数据
- **数据损坏**：JSON 解析失败时返回默认空数组
- **存储容量**：localStorage 通常有 5-10MB 限制，适合文本数据存储

## 内存同步策略

- 组件挂载时（`useEffect`）从 localStorage 读取数据
- 每次增删改操作后立即写入 localStorage
- 数据变更通过 `refresh()` 回调同步 UI
