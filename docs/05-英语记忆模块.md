## 交互式创建

词书和句书不再使用文本域批量导入，改为**交互式创建**。

### 创建流程

```
英语记忆页
├── 📗 创建词书  →  弹窗输入名称  →  跳转到词书详情页（空）
├── 📘 创建句书  →  弹窗输入名称  →  跳转到句书详情页（空）
└── [列表]
    ├── 每本词书 → [✏️编辑] [⬇️导出] [🗑️删除]
    └── 每本句书 → [✏️编辑] [⬇️导出] [🗑️删除]
```

## 详情页（BookDetailView）

点击"编辑"或创建后自动跳转到单本书的详情页。

### 顶部操作栏

- **书名**（大号显示）+ 类型标签（📗 词书 / 📘 句书）
- **＋ 添加词/句** → 弹出窗口，填写英文和中文
- **⬇️ 导出** → 将该书导出为 JSON
- **✏️ 重命名** → 弹窗修改书名
- **🗑️ 删除** → 确认后删除整本书

### 条目卡片

每个词/句以**圆角矩形卡片**展示：

```
┌──────────────────────────────────────┐
│  abandon                    [✏️][🗑️] │
│  放弃                                  │
└──────────────────────────────────────┘
```

- **左上**：英文在上（大号加粗），中文在下（小号灰色）
- **右下**：编辑按钮和删除按钮

### 粘性头部

详情页的顶部（书名 + 操作按钮）使用 `position: sticky; top: 0` 固定，滚动条目列表时头部始终可见，方便随时添加或操作。

### 添加/编辑弹窗

均为模态弹窗，包含两个输入框：

```
┌── 添加词 ──────────────────────┐
│ 英文  [___________________]    │
│ 中文  [___________________]    │
│             [取消] [确认添加]   │
└────────────────────────────────┘
```

编辑弹窗预填当前值，修改后确认。

## 内容选择机制（记忆前）

开始记忆前，用户在选择界面勾选条目：

- **整本书选择**：勾选书旁的复选框，自动选中该书所有条目
- **部分条目选择**：展开书本，单独勾选需要的条目
- **混合选择**：可同时从多个词书和句书中选择条目

## 闪卡记忆系统

重构后的英语记忆采用**间隔重复（Spaced Repetition）**算法。

### 核心算法

```
初始状态：
  queue      = 打乱后的全部待测词条
  retestQueue = []    （答错后待复习的词条）
  answeredCount = 0   （已答题数，用于计算复习时机）

取下一个词（nextWord）：
  1. answeredCount++
  2. 检查 retestQueue 是否有到期的（dueAfter ≤ answeredCount）
     → 有：取出作为当前词
  3. 否则从 queue 取下一个
  4. queue 空但 retestQueue 还有 → 强制清出
  5. 两者皆空 → 完成
```

### 答题状态机

```
                  ┌─────────────┐
                  │   showing   │  ← 显示中文，等待输入
                  └──────┬──────┘
                   正确 │    │ 错误
                  ┌─────▼┐   │
                  │ 完成 │   │
                  └──────┘   │
                             ▼
                  ┌─────────────┐
                  │ wrong-reveal│  ← 显示正确答案，要求重新输入
                  └──────┬──────┘
                   输对了 │
                  ┌─────▼──────────────┐
                  │ 送入 retestQueue   │
                  │ dueAfter = 当前计数 │
                  │           + 3~5   │
                  └────────────────────┘
```

### 比对规则

```typescript
userInput.trim().toLowerCase() === correct.trim().toLowerCase()
```

- 忽略首尾空白
- 忽略大小写差异
