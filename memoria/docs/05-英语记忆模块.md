## 词书 / 句书导入

用户可以在文本框中按行输入，每行格式为：

```
英文内容 = 中文意义
```

支持的分隔符：`=`、`:`、`：`

示例：

```
abandon = 放弃
ability = 能力
able = 能够
```

## 内容选择机制

### 选择方式

- **整本书选择**：勾选词书/句书旁的复选框，自动选中该书所有条目
- **部分条目选择**：展开书本，单独勾选需要的条目
- **混合选择**：可同时从多个词书和句书中选择条目

### 选择状态管理

- 词书和句书各有独立的选中 ID 集合（`Set<string>`）
- 书本全选和部分选择互相同步：
  - 勾选书本 → 所有条目的 ID 加入选中集合
  - 取消书本 → 所有条目的 ID 从选中集合移除
  - 取消某一条目 → 对应书本的全选状态自动取消

## 闪卡记忆系统（v2 重构）

重构后的英语记忆采用**间隔重复（Spaced Repetition）**算法。每次屏幕中央仅显示一个词条的中文意义，用户输入英文后即时反馈。

### 核心算法

```
初始状态：
  queue      = 打乱后的全部待测词条
  retestQueue = []    （答错后待复习的词条）
  answeredCount = 0   （已答题数，用于计算复习时机）

取下一个词（nextWord）：
  1. answeredCount++
  2. 检查 retestQueue 是否有到期的（dueAfter ≤ answeredCount）
     → 有：取出作为当前词
  3. 否则从 queue 取下一个
  4. queue 空但 retestQueue 还有 → 强制清出
  5. 两者皆空 → 完成
```

### 答题状态机

```
                  ┌─────────────┐
                  │   showing   │  ← 显示中文，等待输入
                  └──────┬──────┘
                   正确 │    │ 错误
                  ┌─────▼┐   │
                  │ 完成 │   │
                  └──────┘   │
                             ▼
                  ┌─────────────┐
                  │ wrong-reveal│  ← 显示正确答案，要求重新输入
                  └──────┬──────┘
                   输对了 │
                  ┌─────▼──────────────┐
                  │ 送入 retestQueue   │
                  │ dueAfter = 当前计数 │
                  │           + 3~5   │
                  └────────────────────┘
```

### 详细流程

1. **展示**：屏幕中央大号字体显示中文意义，下方提供输入框
2. **首次答题**：
   - **正确** ✅ → 该词**永久标记为完成**，不再出现
   - **错误** ❌ → 界面变红，显示正确的英文单词，提示用户重新输入
3. **纠错输入**：用户看到正确答案后，在输入框中输入正确单词。只有当输入完全正确时才能继续
4. **间隔复习**：纠错正确后，该词进入 retest 队列，设定 `dueAfter = 已答题数 + random(3, 5)`。即再经过 3~5 个其他单词后，该词会重新出现
5. **复习时**：
   - 该词重新以中文意义的形式出现
   - **正确** ✅ → 最终完成，不再出现
   - **错误** ❌ → 回到步骤 3，再次进入 retest 队列

### 界面说明

- **顶部进度条**：显示 `已完成/总数 · 错误次数`，带进度百分比条
- **中央卡片**：
  - 新词标注 `📝 新词`，复习词标注 `🔁 复习`
  - 中文释义以大号字体居中显示
  - 输入框自动聚焦，支持 Enter 快速提交
- **错误反馈**：卡片变红，显示正确答案，用户必须重新输入正确内容
- **完成页**：展示总题数、答对次数、曾答错次数、最终正确率，以及错词回顾列表

### 比对规则

```typescript
userInput.trim().toLowerCase() === correct.trim().toLowerCase()
```

- 忽略首尾空白
- 忽略大小写差异
